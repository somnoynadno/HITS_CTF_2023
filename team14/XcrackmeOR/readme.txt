Сей crackme написан на C++, без использования обфускаторов/antiVM/antiDebug

Features:
Динамическая генерация ключа, т.е. ключ генерируется при вводе имени, если ввести правильный ключ, высветится флаг
Флаг статичен, однако через просмотр констант в дебаггере полностью не отсвечивается, т.к. составляется посимвольно
При вводе неправильных символов имени, приложение тебя буллит
При запуске в Ida Pro высвечивается окно, что есть ссылки на файлы дебага, однако в пути написано Debug files is no Help,
что как бы должно давать понять, что оно не поможет

Принцип работы:
Принцип работы такой, что каждый символ имени xor'ится с 1010101, это и есть ключ, при вводе которого, мы получаем флаг
Алгоритм XOR'a такой, что каждый символ переводится в двоичный int согласно таблице ASCII, дальше XOR с 0b1010101

Пути решения:
Для решения можно применить дебаггер, отследив то, что происходит с памятью 
Либо брутфорсом, введя имя в один символ и перебрав все символы таблицы ascii
upd: решить дебаггером у меня не получилось, поэтому добавил строку с подсказкой
(1 c0u1d n0t f1nd key f0r f1ag, maybe you can do it?)
В которой зашифровано 1010101, с которым посимвольно xor'ится строка имени
